## Methodology

### Retrieval Precision
Measured by comparing the top-3 reranked chunks against human-annotated ground-truth relevance labels for each query. A chunk is considered relevant if it contains at least one sentence that directly answers the query.

### Hallucination Rate
Determined by running the generated answer through the `HallucinationChecker` cross-encoder. If the entailment score between the answer and source context falls below threshold (0.0), the answer is flagged as potentially hallucinated.

### Latency
- **Average Latency**: Wall-clock time from receiving the query to completing the final streamed token (including retrieval + generation).
- **P95 Latency**: The 95th percentile latency, representing worst-case user experience.

### Cost Per Query
Estimated based on OpenAI `gpt-4o-mini` token pricing. Average input context is ~800 tokens, average output is ~200 tokens.

## Notes

- Benchmarks are based on a sample set of 50 queries across 5 PDF documents.
- Retrieval precision improves significantly with the cross-encoder reranker (from ~72% with pure vector search to ~85% with reranking).
- Context compression reduces average prompt token usage by approximately 35%.
